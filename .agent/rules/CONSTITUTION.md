# Antigravity 開発憲法

## 1. 思考と言語の原則
- **思考プロセス**: 内部的な推論は英語で行う。
- **ユーザー応答**: 回答、技術サマリー、ボタン文言はすべて自然な**日本語**で行う。
- **プッシュ前確認**: 人間がプッシュする前に、必ず「何を変更し、何を確認したか」のサマリーを提示すること。

## 2. UI/UX 設計の黄金律
- **脱・テーブル**: 店舗リストに Markdown テーブル（`|---|`）は絶対に使用しない。必ず `<div class="store-card">` 形式で出力すること。
- **レスポンシブGrid**: CSS Grid を用い、スマホ1列・PC3列のレイアウトを維持する。
- **店名正規化**: 「ジャンカラ ジャンカラ梅田店」のような重複を自動削除し、「ジャンカラ 梅田店」とすること。

## 3. 収益化（アフィリエイト）の保護
- **広告コードの不可侵**: `inline_ad_html` と `sticky_footer_html` はサイトの生命線である。削除・改変は厳禁。
- **Hugoエスケープ対策**: Markdown内にHTMLを書き出す際は、行頭のインデントを一切行わず（0文字目開始）、前後に空行を挿入すること。

## 4. 実行の誠実さ
- 「修正した」と報告する前に、必ず `page_generator.py` を実行し、生成された `.md` ファイルの中身を物理的に確認すること。
- 以前のコードが残っている場合は、付け足しではなく「完全な削除と上書き」を行うこと。
